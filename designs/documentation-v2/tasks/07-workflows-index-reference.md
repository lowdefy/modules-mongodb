# Task 7: Workflows `index.md` + `reference/` (Phase 3a)

## Context

Workflows is the most concept-heavy module and the **exemplar** that drives the whole structure ‚Äî it's authored in full before the other ten modules follow. This task builds its `index.md` and the lookup-oriented `reference/` pages. The concepts (Task 8) and how-to (Task 9) are separate tasks. The var table is generated by Task 3's `gen-var-docs.mjs`.

Reference pages are for *looking up* ‚Äî what vars exist, what's exported, what the FSM states are, what form components exist, the authoring grammar, the required indexes.

## Task

Author `docs/workflows/index.md` and `docs/workflows/reference/*`. Every file gets front-matter (`module: workflows`; `type: index` for index, `type: reference` for the rest).

**`index.md`** (`type: index`) ‚Äî one-paragraph description, a dependencies table, "when to use", a single worked `lowdefy.yaml` quickstart snippet (how a consumer adds the workflows module), and links to the module's concept/how-to/reference pages and to relevant `docs/shared/` idioms.

**`reference/vars.md`** ‚Äî **generated** by `node scripts/gen-var-docs.mjs` (do not hand-author). Just ensure the generator has been run and the output is correct for workflows (it has deeply nested vars ‚Äî verify the nested rendering reads well).

**`reference/exports.md`** ‚Äî hand-written list of the module's exports (pages, components, API endpoints, connections, menus) from `modules/workflows/module.lowdefy.yaml`.

**`reference/indexes.md`** ‚Äî required MongoDB indexes, mined from the current `modules/workflows/README.md` index requirements, **plus the `actions` validator constraint** the design calls out.

**`reference/fsm-and-signals.md`** ‚Äî the FSM state/signal **tables**, mined from `designs/workflows-module-concept/state-machine/`. This is the lookup companion to the `concepts/signals-vs-status.md` explanation (Task 8) ‚Äî tables here, narrative there.

**`reference/form-components.md`** ‚Äî form component reference, mined from `modules/workflows/components/fields/README.md`.

**`reference/authoring-grammar.md`** ‚Äî the action-authoring grammar, mined from the action-authoring sections of the current `modules/workflows/README.md`.

## Acceptance Criteria

- `docs/workflows/index.md` exists with index front-matter, a dependencies table, when-to-use, a working `lowdefy.yaml` quickstart snippet, and links to concepts/how-to/reference + relevant `docs/shared/` idioms.
- `docs/workflows/reference/vars.md` is the generated output (carries the generated marker) and renders workflows' nested vars readably.
- `exports.md`, `indexes.md`, `fsm-and-signals.md`, `form-components.md`, `authoring-grammar.md` exist with `type: reference` front-matter and content migrated from the cited sources.
- `indexes.md` includes the `actions` validator constraint, not just index definitions.
- All internal links resolve; `pnpm docs:check` passes (front-matter valid, llms.txt regenerated if run).
