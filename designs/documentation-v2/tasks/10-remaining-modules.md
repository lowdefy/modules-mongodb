# Task 10: Migrate the Remaining Ten Modules (Phase 4)

## Context

The workflows exemplar (Tasks 7–9) proved the structure end-to-end. This task migrates the other ten modules — `activities`, `companies`, `contacts`, `events`, `files`, `layout`, `notifications`, `release-notes`, `user-account`, `user-admin` — into `docs/{module}/` trees, then reduces each `modules/{name}/README.md` to a stub.

Unlike workflows, these are all small — a single README has served each adequately to date — so most collapse to an `index.md` plus a generated `reference/vars.md` (and `reference/exports.md` where it helps). `concepts/` and `how-to/` are the **exception**, added only where a module actually has non-obvious material. The per-module subfolder call is **delegated to you, the implementer, at authoring time** — the modules are simple enough that the judgment is cheap and low-risk; do not manufacture empty `concepts/`/`how-to/` folders for symmetry (design principle: build for concrete needs, not speculation).

## Task

For **each** of the ten modules:

**1. `index.md`** (`type: index`, required) — one-paragraph description, dependencies table, "when to use", a single worked `lowdefy.yaml` quickstart snippet, and links to the module's reference pages and to relevant `docs/shared/` idioms. Mine the content from the module's current `modules/{name}/README.md`. Where the README links `docs/idioms.md#anchor`, link the corresponding `docs/shared/*.md` instead (these are the links the design deferred from Task 6).

**2. `reference/vars.md`** — generated by `node scripts/gen-var-docs.mjs` (do not hand-author). Run the generator; it produces output for every module with vars.

**3. `reference/exports.md`** — hand-written, **where it helps** (modules with a non-trivial export surface). Mine from the manifest.

**4. `concepts/` / `how-to/`** — only where the module has genuinely non-obvious material in its current README. Most modules will have neither. Use judgment.

**5. Fold and delete `modules/activities/VARS.md`** — the last surviving legacy VARS file. Its content goes into `docs/activities/` (the generated `vars.md` covers the var schema; any prose goes in `index.md`), then delete the file. Note it currently links `idioms.md` — that link dies with the file.

**6. Reduce each `modules/{name}/README.md` to a stub** — a short paragraph pointing at the central docs, e.g. "Full docs: [`../../docs/{name}/`](../../docs/{name}/index.md)". This is where the 9 module READMEs' `idioms.md#anchor` links vanish (Task 6 deferred them here).

## Acceptance Criteria

- Each of the ten modules has `docs/{module}/index.md` with index front-matter and migrated content; idiom links point at `docs/shared/*.md`, not `idioms.md`.
- `docs/{module}/reference/vars.md` is generated (carries the generated marker) for every module with vars.
- `concepts/`/`how-to/` exist only where a module has real non-obvious material — no empty placeholder folders.
- `modules/activities/VARS.md` is deleted and its content folded into `docs/activities/`.
- Every `modules/{name}/README.md` (all ten) is a stub linking into `docs/{name}/`.
- `grep -rn "idioms.md" modules/*/README.md modules/*/VARS.md` returns nothing.
- `pnpm docs:check` passes (front-matter valid, var tables current, llms.txt regenerated if run).
- `pnpm ldf:b` still compiles.

## Files

- `docs/{module}/index.md` × 10 — create.
- `docs/{module}/reference/vars.md` × (modules with vars) — generated.
- `docs/{module}/reference/exports.md` — create where it helps.
- `docs/{module}/concepts/*` , `docs/{module}/how-to/*` — create only where warranted.
- `modules/{name}/README.md` × 10 — modify → stub.
- `modules/activities/VARS.md` — delete (content folded into `docs/activities/`).

## Notes

- The modules are deliberately under-specified here per the design: the subfolder shape is your call at authoring time. When unsure, prefer fewer files (`index.md` + generated `vars.md`) over speculative `concepts/`/`how-to/`.
- `modules/shared/` is **not** a module — no `docs/shared/{module}` folder for it (`docs/shared/` is the idioms folder from Task 6, a different thing).
- Run the var generator once at the end so every module's `vars.md` is current before `docs:check`.
