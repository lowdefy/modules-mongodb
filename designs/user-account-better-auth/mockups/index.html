Design system

The same language as user-admin — two chromes

Ant Design (compact) on the repo's violet accent, reusing the user-admin deck's tokens verbatim so the two modules read as one product. This module renders in two chromes: the signed-in account workspace lives in the layout app shell; the eight public auth pages use the real layout/auth-page component — a centred 360px card with a primary-gradient cover band over the logo. Nothing here invents a palette; the risk budget goes to the method-driven login and the settings-row workspace.

Method-driven login

Settings-row tiles

Gradient avatar

Write-pathway honesty

Contact fields → app request. Password / 2FA / passkeys / sessions → BetterAuth client actions. Each section saves on its own.

Page · login · signature

One page, rendered from the methods var

The login page shows only the sign-in methods the deployment enables (Decision 2). Same file, same chrome — three deployments below prove it. All methods dispatch through the one Login action; BetterAuth returns errors inline, so the hard-wall and unverified-email states render in-page instead of NextAuth's dead ?error= redirect.

Option A✓ Chosen direction

Account workspace — sections that save themselves

One view page replaces today's view + edit (Decision 5). Each section edits through its own modal and its own write pathway, so a profile save (a change-stamped contact request) never rides the same Save button as half a dozen session-gated auth calls. Profile and Security settings on the left; Sessions and Linked accounts on the right. Security uses a settings-row grammar rather than admin's description lists — this is the person editing their own account, not an operator reading someone else's.

Option B

Separate tiles per concern — alternative, not chosen

Resolves the design's open question the other way: Profile / Password / Two-factor / Passkeys / Sessions / Linked accounts as six sibling tiles in a masonry grid. Honest to the write pathways (each tile = one routine), but noisier — every security control demands equal weight, and the page stops fitting one glance. Chosen A folds password/2FA/passkeys/linked into one Security tile because they're all "how you sign in," and gives Sessions its own tile because it's the one thing you act on often.
