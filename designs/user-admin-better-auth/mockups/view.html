User workspace — Option A (classic two-column)

One detail page replaces today's view + edit + check. Main column: Profile, Access, Global attributes (each edits through its own modal + routine). Sidebar: a single Security tile that now owns every access-lifecycle and auth control — including the cross-app badges — plus an Activity timeline.

  • No destructive actions in the title-block. Suspend / Remove / Delete all live in the Security card, co-located with the Apps list that determines their blast radius and availability.
  • Profile → contact request (change-stamped). Access → UpdateMemberRoles + UpdateMemberAttributes. Global attributes → UpdateUserAttributes.
  • Suspend → BanUser (global, reversible, revokes sessions) — see confirm dialog below. Remove → RemoveMember. Sign out everywhere → RevokeUserSessions. Delete → DeleteUser, gated on zero other memberships.
  • Apps badges + Sessions + Auth methods are native reads (user-members+user-organizations, user-sessions, user-accounts). Apps is read-only visibility.
