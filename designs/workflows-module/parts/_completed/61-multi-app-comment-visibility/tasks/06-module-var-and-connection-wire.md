## Task

### `modules/workflows/module.lowdefy.yaml`

Add a new module var under the `vars:` block (place it near `app_name`):

```yaml
enable_internal_comments:
  type: boolean
  default: false
  description: >
    Opt-in per app: when true, this app's comment surfaces show a shared/internal
    toggle next to every comment input, and the engine honours an `internal`
    comment (written into the submitting app's display bucket only). When false
    (the default), the toggle is hidden and the engine coerces any `internal`
    request to `shared`, so every comment from this app is visible to all apps
    that already see the event. Single-app and customer-facing apps leave this
    off; the internal/team app in a multi-app deployment sets it true. Wired onto
    the workflow-api connection (where the engine enforces it) and read at build
    time by the comment surfaces (to show/hide the control).
```

### `modules/workflows/connections/workflow-api.yaml`

Add the connection property, wired from the module var (place it near `app_name`):

```yaml
enable_internal_comments:
  _module.var: enable_internal_comments
```
