- `docs/workflows/reference/vars.md` — generated; currently has an `entities` row.

## Task

### `docs/workflows/index.md`

- In the `lowdefy.yaml` vars example, remove the `entities:` map (lines 43-51). The example's module `vars` block should show only `workflows_config` and `app_name` (matching the demo's `vars.yaml` after Task 5).
- Remove the `entities` vars bullet (line 56). The `entity:` block is per-workflow authoring config, not a module var, so it does not belong in this module-vars list — but add a short pointer noting that each workflow declares its entity wiring (connection_id, ref_key, page link) in an `entity:` block, cross-linking to `reference/authoring-grammar.md`.
- The "See `apps/demo/modules/workflows/vars.yaml`" pointer (line 58) stays valid.

### `docs/workflows/reference/authoring-grammar.md`

- Replace the flat `entity_collection`/`entity_ref_key` entries (lines 16-17) with the `entity:` block:

  ```yaml
  entity: # required — the workflow's entity wiring
    connection_id: <slug> # required — Lowdefy connection id for the entity (e.g. leads-collection)
    ref_key: <key> # required — event-references key (e.g. lead_ids)
    page_id: <page> # required — host-app page id the back-link navigates to
    id_query_key: <key> # optional — URL query key for the entity id (default _id)
    title: <Label> # required — singular entity-kind label (e.g. Lead)
  ```

- If the grammar doc has a fuller field reference table or section elsewhere, add the five `entity.*` fields there too with the same required/optional/default semantics.

### `docs/workflows/reference/vars.md`

- Regenerate via `pnpm docs:gen` (do not hand-edit). The `entities` row drops out and the `workflows_config` description reflects Task 4's rewrite.
