# Docs contributing guide

Every file in `docs/` must open with a YAML front-matter block. The linter (`scripts/gen-llms-txt.mjs`, run via `pnpm docs:check`) validates this schema at CI time.

## Front-matter schema

```yaml
---
title: Signals vs Status # required — human-readable page title
module: workflows # required — see values below
type: concept # required — see values below
concepts: [signals, fsm, status] # optional — string array of key concepts covered
---
```

### `module`

The module this file belongs to, or one of the special values:

| Value           | Meaning                                                          |
| --------------- | ---------------------------------------------------------------- |
| `root`          | Repo-level docs (index, this file)                               |
| `shared`        | Cross-cutting idioms under `docs/shared/`                        |
| `plugins`       | Plugin package docs under `docs/plugins/`                        |
| _(module name)_ | A module under `modules/` — e.g. `workflows`, `layout`, `events` |

### `type`

| Value       | When to use                                                         |
| ----------- | ------------------------------------------------------------------- |
| `index`     | Root index or module landing page                                   |
| `concept`   | Explains a concept — what it is, why it exists, how it works        |
| `how-to`    | Goal-oriented guide — step-by-step instructions for a specific task |
| `reference` | Exhaustive enumeration — vars, exports, API shapes, option tables   |
| `shared`    | Cross-module idiom docs living under `docs/shared/`                 |

### `concepts`

Optional. A flat string array of the key concepts or terms the page covers. Useful for search and for the `docs/llms.txt` index (generated — do not hand-author).

## File placement

```
docs/
  index.md              — type: index, module: root
  CONTRIBUTING.md       — type: reference, module: root
  shared/               — cross-cutting idiom pages
  workflows/            — workflows module docs
  plugins/              — plugin package docs
```

Per-module subdirectories use three sub-types:

```
docs/{module}/
  index.md              — type: index
  concepts/             — type: concept
  how-to/               — type: how-to
  reference/            — type: reference
```

## What not to do

- Do not hand-author `docs/llms.txt` — it is generated by `scripts/gen-llms-txt.mjs` (`pnpm docs:gen`).
- Do not invent links to doc pages that do not exist — link only to pages already present under `docs/`. (Source-side `modules/{name}/README.md` files are stubs; link to the corresponding `docs/{module}/` page instead.)
- Do not add front-matter fields not listed here without updating this file first.
